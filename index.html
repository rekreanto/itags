It is suboptimal from a langauge usability point-of-view if the langauge user/programmer:

  1. Has to shift frequentlys between representations/languageages when working on an artefact
  2. Is punished with incidencial complexity for composing an artefact out of parts
  3. Must use expresseions whose consequences must kept in mind and addressed in a far-away place
  4. Has no clear way to get a birds-eye view of an application
  5. etc. etc.

Itags addresses all those concerns and does so at the appropriate abstraction level, before the problems already has escaped and caused havoc in the souce code and permanent damage to the language usage experience.

The key to the Itags solution is the following points

  • Orthogonality at the core (1)
  • Data as a universal interface (2)
  • Ephemeral language constructs that knows when to die (3)
  • Uses an explicit state formalism; Layer Machines (4)
